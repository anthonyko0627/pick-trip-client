# 테스트 규칙

저장소는 Vitest를 단위/컴포넌트 테스트 러너로 사용한다. DOM 환경(jsdom)과 React Testing Library, `@testing-library/jest-dom` 매처가 설정되어 있다. 테스트 파일은 검증 대상 소스 옆에 `*.test.ts`, `*.test.tsx` 형식으로 둔다.

## 기본 검증

```bash
bun run lint
bun run build
bun run test
```

PR 제출 전 세 명령어를 모두 실행한다.

## 테스트 실행

```bash
bun run test          # 1회 실행
bun run test:watch    # watch 모드
bun run test:coverage # 커버리지 측정
```

## 테스트 도구 추가 기준

테스트 도구를 도입할 때는 다음 우선순위를 따른다.

| 목적 | 권장 도구 |
| --- | --- |
| 유틸 함수, 타입 변환 | Vitest |
| 컴포넌트 렌더링 | React Testing Library |
| 핵심 사용자 흐름 | Playwright |

도구 추가 PR에는 다음 내용을 포함한다.

- 추가 이유
- 실행 명령어
- CI 연동 방식
- 예시 테스트 1개 이상

## 테스트 작성 원칙

- 사용자가 보는 동작을 기준으로 검증한다.
- 구현 세부사항보다 입력, 화면 상태, API 호출 결과를 검증한다.
- 날짜, 지역 코드, API 응답 변환처럼 회귀 위험이 큰 로직은 단위 테스트를 우선한다.
- 일정 생성, 저장, 공유처럼 핵심 플로우는 E2E 테스트 후보로 둔다.

## 테스트 데이터

- 실제 secret, 실제 사용자 토큰, 운영 API를 테스트에 사용하지 않는다.
- API 응답 fixture는 최소 필드만 포함하되, 오류 응답도 함께 준비한다.
- 시간 의존 로직은 고정된 날짜를 사용한다.

## 수동 검증 체크리스트

- 모바일 너비에서 콘텐츠 목록, 선택 목록, 일정 결과가 겹치지 않는다.
- 로딩, 빈 목록, 오류 상태가 표시된다.
- 인증이 필요한 액션에서 로그인 유도 흐름이 동작한다.
- 브라우저 콘솔에 불필요한 오류가 없다.
